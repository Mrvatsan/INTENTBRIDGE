# Contributing to IntentBridge

Thank you for your interest in contributing to IntentBridge! As a modular intent processing system, we value contributions that improve the precision, stability, and transparency of the pipeline.

## Contribution Guidelines

1. **Modular Design**: Ensure any changes respect the decoupling between Module 1 (Classification) and Module 2 (Resolution).
2. **Transparent Logic**: All assumptions or classification heuristics should be explicitly documented and neutral.
3. **No Executable Changes in Docs**: Keep documentation updates strictly non-functional.
4. **Clean Commits**: Follow the `type: description` commit message format.

## Development Setup

1. Fork the repository.
2. Create a new branch for your feature or fix.
3. Install dependencies: `pip install -r requirements.txt`.
4. Run the app: `python app.py`.

We look forward to your pull requests!
